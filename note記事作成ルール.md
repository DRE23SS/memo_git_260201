# note記事作成ルール

このルールは、AIに記事作成を依頼する際のプロンプト/指示書として使用します。
以下の原則とステップに従って、読者に「伝わる」記事を作成してください。

---

## 📌 基本原則

### 原則1: 伝えたいことより「伝えられたいこと」を書く

- **「伝える」と「伝わる」は別物**
  - 伝える＝情報を投げかける（書き手主体）
  - 伝わる＝読者が理解・納得・共感する（読み手主体）
- **読者の「困りごと」を起点に考える**
  - 誰に → どんな気持ちになって → どんな行動をしてほしいか

> 例：「AIツールの使い方」を書くなら  
> ❌ 自分が知っていることを羅列  
> ✅ 「AIを使いたいけど何から始めればいいかわからない」という困りごとを解決する

---

### 原則2: ひと言でまとめる

- **20文字以内で、小学生にもわかる言葉で核心を伝える**
- **1つの文に1つのメッセージのみ**
- **「つまり」と問いかけて、ひと言になるまで繰り返す**

> 例：  
> ❌ 「本記事ではAI時代における効率的なコンテンツマーケティング手法について解説します」  
> ✅ 「AIに記事を書かせる3つのコツ」

---

### 原則3: O.R.E.O公式で構成する

1. **Opinion（意見）**: 主張を述べる
2. **Reason（理由）**: なぜそう言えるのか
3. **Example（具体例）**: 実際の例を示す
4. **Opinion（意見）**: 主張を強調して締める

> 例：  
> O：週4日勤務制を導入すべきだ  
> R：社員のモチベーションが上がり、収益もアップする  
> E：ハイブリッドワークで成果を上げている企業が多い  
> O：10月から段階的に導入を進めよう

---

### 原則4: 抽象を捨て、定量で語る

- **曖昧な言葉を避け、具体的な数字・事実を使う**
- **「すごい」「やばい」→ 何がどうすごいのか分解する**

> 例：  
> ❌ 「かなり成果が出ました」  
> ✅ 「PV数が前月比150%に増加しました」

---

### 原則5: 読者視点でポジティブに変換する

- **相手を楽しませる言い方を考える**
- **ネガティブな情報も、ポジティブな視点で伝えられないか検討する**

> 例：  
> ❌ 「あいにくの雨ですね」  
> ✅ 「雨のおかげで、ゆっくりお話できますね」

---

## 🎨 読みやすさのデザイン（10のポイント）

読みやすい文章は「デザイン」されている。  
**「読まなくても、読める」** 状態を目指す。

### Point 1: 一段落は3〜4行にする

- 長い文章の塊は、どこから読むべきかわからなくなる
- 短すぎる1行だけの段落が続くのもNG
- **3行以上の高さがあると「読みやすい」と感じる**

---

### Point 2: 一文は1〜2行以内にする

- 文が長いと主語と述語の対応がわかりづらくなる
- 長くても3行までにおさめる

> 例：  
> ❌ 「1つの段落は3〜4行以内に収めるようにしていますが、これは一体なぜかというと、長い文章が大きな塊になっているとどこから読むべきか分からなかったり、重要なポイントが掴みにくくなるからです。」  
> ✅ 「1つの段落は、3〜4行以内に収めるようにしています。長い文章が大きな塊になっていると、どこから読むべきか分からなかったり、重要なポイントが掴みにくくなります。」

---

### Point 3: 漢字よりもひらがなを使う

- **1行あたりの漢字は10字程度に調整**
- 読み手の引っかかりを減らす

| 漢字   | ひらがな |
| ------ | -------- |
| 例えば | たとえば |
| 結構   | けっこう |
| 等     | など     |
| 所謂   | いわゆる |
| お勧め | おすすめ |

---

### Point 4: わかりやすい言葉を使う

- 複数の表現があるときは、もっとも一般的な言葉を選ぶ
- 「これってどういう意味？」と引っかかる言葉はノイズになる

| 難しい表現   | わかりやすい表現 |
| ------------ | ---------------- |
| 微塵も       | 少しも           |
| 面妖な       | ふしぎな         |
| 寂寥感がある | さびしさを感じる |

---

### Point 5: 主語・述語をはっきりさせる

- **「AはBです」の形がはっきり見えるように**
- 指示語（これ、それ）だけでなく、具体的な内容まで示す

> 例：  
> ❌ 「さきほどの内容に似ていますが」  
> ✅ 「さきほどの『Point 3：漢字よりもひらがなを使う』に似ていますが」

---

### Point 6: 重要なポイントに太字（ハイライト）

- 視覚的に「ここが重要」と示す
- 長めの文章でも、注目すべき箇所がわかりやすくなる

---

### Point 7: 重要なキーワードに「かぎ括弧」

- 長い部分には太字、短いキーワードにはかぎ括弧
- **タイトルで目立たせたいキーワードにも効果的**

> 例：  
> ・ワイヤーフレームは"しゃべりながら"つくる  
> ・Webデザインで「紙っぽさ」を出す方法を考えてみる

---

### Point 8: 読み飛ばすための「余白」をつくる

- 重要なことだけが高密度で書かれているより、適度に読み飛ばせる文章があるほうがラク
- **具体例を多めに入れる**

> 例：  
> ❌ 「私が言う『紙っぽさ』とは、おもに『雑誌っぽさ』のことです。」  
> ✅ 「私が言う『紙っぽさ』とは、おもに『雑誌っぽさ』のことです。Webデザインだけど、雑誌っぽい。ディスプレイに映っているけれど、エディトリアル感がある。」

---

### Point 9: 「見出しだけ」で読めるようにする

- 目次を見たときに「自分が読みたいパートはここだ！」とわかるように
- **見出しで内容がわかりづらければ、言葉選びを見直す**

---

### Point 10: 「画像だけ」で読めるようにする

- 文章を読むのは大変な行為
- **参考サイトにはスクリーンショットを添える**
- 「こんな話をしています」「こんな気持ちになっています」が一目で伝わる画像を入れる

## 📝 記事作成ステップ

### Step 1: ターゲットと困りごとを明確にする

```
【ターゲット】
- 誰に向けた記事か？（具体的なペルソナ）
- その人はどんな状況にいるか？

【困りごと】
- どんな悩み・課題を抱えているか？
- 何を解決したいと思っているか？
```

---

### Step 2: メッセージを決める（50〜80文字）

```
【メッセージの公式】
「〈よさ〉だから〈わけ〉だ」

【チェック項目】
□ 読者目線で「そうかも！」と思えるか
□ 既知でも未知でもなく「伏知」（気づいていなかった価値）か
□ 20文字以内で核心を言えるか
```

---

### Step 3: 構成を組み立てる（同分解展）

| フェーズ | 内容               | 例                                                       |
| -------- | ------------------ | -------------------------------------------------------- |
| **同意** | 悩みや課題の指摘   | 「AIに記事を書かせても、なんか薄い内容になりませんか？」 |
| **分析** | 問題点の背景を分析 | 「それは"自分の文体"をAIに教えていないからです」         |
| **解決** | 解決策の提示       | 「3ステップで自分のコピーを学習させる方法があります」    |
| **展望** | 解決による効果     | 「これで量産しても"あなたらしい"記事が書けます」         |

---

### Step 4: 本文を書く

```
【文章のルール】
□ 1文1メッセージ（一文は1〜2行以内）
□ 1段落は3〜4行にする
□ 主語と述語を離しすぎない
□ 重複する言葉は言い換えor省略
□ 「と思う」「と感じる」を使わず言い切る
□ 余計な接続詞・敬語・修飾語を削る
□ 漢字よりひらがな（1行10漢字目安）
□ 箇条書きを活用してわかりやすく
```

---

### Step 5: 書き出しを工夫する

以下のパターンから選択：

1. **よかった要素を描写する**
   - 何について書くかを宣言するように始める

2. **自分語りをする**
   - 自分の体験から始めて、共通項に持っていく

3. **文脈で始める**
   - 「今○○が話題だけど、実は△△との共通点がある」

4. **問いで始める**（頻用しない）
   - 「なぜ○○なのか？」という疑問で引き込む

---

### Step 6: 視覚的な仕上げ

```
【デザインチェック】
□ 重要なポイントに太字を使っているか？
□ キーワードに「かぎ括弧」を使っているか？
□ 見出しだけで内容がわかるか？
□ 適切な画像・スクリーンショットを入れたか？
□ 読み飛ばせる余白があるか？
```

---

### Step 7: 最終チェック

```
【チェックリスト】
□ 冒頭に概要を書いているか？
□ 一文一義で書いているか？
□ 読者の視点で書いているか？
□ 重要なことから書いているか？
□ 文はこれ以上短くならないか？
□ 曖昧な表現はないか？
□ 係り受けは明確か？
□ 否定文は肯定形で書けないか？
□ 「そうかも！」と思える発見があるか？
```

## 🎯 説得力を高めるテクニック

### スパイシーサンド法

**強烈な提示 ＋ なぜ？ ＋ 結論**

> 例：  
> ！「病院のデータ活用はなぜ遅れているのか」  
> ？「現場の負担増加と専門人材不足が背景にある」  
> 。「結論：現場支援型のDXが急務だ」

---

### 三感法

- **共感**: IとYouをつなぐもの（読者の気持ちに寄り添う）
- **実感**: IとYouの利益になるもの（具体的なメリット）
- **快感**: 脳は短い言葉で分かり合えると喜ぶ（簡潔さ）

---

### たとえ話で言い換える

```
「○○とは△△のようなものだ」

例：「言語化力とは、無意識の氷山を水面に引き上げる力だ」
```
