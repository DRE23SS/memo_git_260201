# Antigravity: MV Lyric Generator

> [!abstract] 概要
> 動画と歌詞を入力するだけで、タイミングとシーンに合わせたお洒落なテロップを自動挿入し、編集可能な「タイムコード」や「カスタム演出ライブラリ」を出力するプロンプト。

## 使い方
1. 以下のプロンプトをAI（ChatGPT / Claude / Gemini等）に貼り付ける。
2. 「承知しました」と返ってきたら、「動画ファイル」と「歌詞全文」を渡す。

## 2.5. 特殊文字・絵文字の取扱い (Emoji & Symbols)

## 3. 重要技術要件：確実な座標指定 (Deterministic Positioning)
**【最重要】MoviePy等のライブラリを使用する際の絶対ルール**

「中央寄せ ('center')」や「スライドイン (SlideIn)」といった**ライブラリの自動配置・簡易エフェクト機能は使用禁止**とします。これらはしばしば競合し、意図しない挙動（中央に固定される、エフェクトが効かない）を引き起こします。

**解決策:「完全計算式 (Pure Math)」アプローチ**
すべての配置と動きは、**整数 (Integer)** の座標計算式として実装してください。

- **不確実な実装 (NG)**:
    ```python
    # ライブラリのブラックボックスな機能に頼る
    txt_clip.with_position('center').with_effects([vfx.SlideIn(0.5)])
    ```
- **確実な実装 (OK)**:
    ```python
    # 毎フレームの座標を厳密に計算する関数を定義
    def pos_func(t):
        progress = t / 0.5
        x = start_x + (target_x - start_x) * progress
        return (int(x), int(target_y)) # 必ず整数タプルを返す

    txt_clip.with_position(pos_func)
    ```

## 3.5. フォント戦略 (Font Strategy)
**以下のGoogle Fontsを優先的に使用して、シーンの空気を演出してください。**
※インストールされていない場合は、（）内の標準フォントにフォールバックします。

1. **基本・元気 (Default/Pop)**:
   - **`Potta One`** (代用: HG創英角ポップ体)
   - 用途：明るい歌詞、掛け声
   - キーワード: Energy

2. **安心・優しさ (Gentle/Round)**:
   - **`Zen Maru Gothic`** (代用: HG丸ゴシックM-PRO)
   - 用途：メインの歌詞、柔らかい表現
   - キーワード: Round

3. **重量感・衝撃 (Heavy/Impact)**:
   - **`Dela Gothic One`** (代用: HGゴシックE / 創英角ゴシックUB)
   - 用途：タイトル、強調、「ドスン」等の擬音
   - キーワード: Heavy

4. **真面目・伝統・締め (Serious/Serif)**:
   - **`Shippori Mincho`** (代用: HG明朝E / MS明朝)
   - 用途：作者名、Fin、和風/シリアス
   - キーワード: Serif

5. **手書き・エモ (Handwriting)**:
   - **`Yusei Magic`** (代用: HGP教科書体 / 手書き風フォント)
   - 用途：日常、思い出、落書き
   - キーワード: Hand

6. **デジタル・レトロ (Pixel)**:
   - **`DotGothic16`** (代用: MSゴシック)
   - 用途：ゲーム画面、エラー、電脳
   - キーワード: Pixel

7. **和風・筆文字 (Brush)**:
   - **`Yuji Syuku`** (代用: HG行書体)
   - 用途：和風、ホラー、決意
   - キーワード: Brush

**※ 計画段階で「このシーンにはこのフォントがベストか？」を常に自問し、必要であればリスト外のフォントも積極的に提案すること。**

## 3.5.5. 文字装飾と配色戦略 (Decoration & Color Details)
**フォント選びと同様に、文字の「装飾（縁取り・色）」も視認性と印象を左右します。以下のプリセットを基準に指定してください。**

1. **基本スタイル (Standard)**:
   - **配色**: 白文字 + 黒縁取り
   - **縁取り幅 (Stroke)**: 中 (2〜3px)
   - **用途**: 通常の歌詞、説明文。最も視認性が高い。

2. **ポップ/元気 (Pop Style)**:
   - **配色**: 明るい色（オレンジ、黄色、水色など） + 濃い同系色または黒縁取り
   - **縁取り幅**: 太め (4〜6px)
   - **用途**: サビ、強調したいフレーズ、元気な挨拶。

3. **重量感 (Heavy Style)**:
   - **配色**: 赤、黒、金 + 白の二重縁取り（可能なら）または極太黒縁取り
   - **縁取り幅**: 極太 (6〜8px)
   - **用途**: 「ドスン」、衝撃音、叫び。

4. **綺麗・繊細 (Clean/Serif Style)**:
   - **配色**: 白、ペールトーン + 縁取りなし（または微細な光彩/1px）
   - **縁取り幅**: なし 〜 細 (0〜1px)
   - **用途**: 作者名、切ない歌詞、静かなシーン。

**【色のルール】**
- 背景が明るい場合：濃い色の文字 または 強めの縁取りを使う。
- 背景が暗い場合：明るい色の文字（白、黄色）を使う。
- **禁止事項**: 文字色と縁取り色を同じ（または近い色）に設定してはならない（例：白文字×白枠）。

## 3.7. カスタム演出ライブラリ (Custom Effects Library)
**複雑な文字演出は `mv_effects.py` として外部ライブラリ化しています。**
このスクリプトが存在しない場合、以下の記述に基づいて新規作成してください（Python/MoviePy）。

**[基本ルール]**
1. スクリプト冒頭で `from mv_effects import ...` を行うこと。
2. ライブラリ関数はスタイル（色・枠）の解釈を行わない。呼び出し側で具体的な数値を渡すこと。

**[エフェクト一覧]**
- **`make_bouncing_text`**: 右から1文字ずつ放物線を描いて飛び込んでくる演出（Pyon Pyon）。元気なシーンに最適。
- **`make_dropping_text`**: 上から1文字ずつ落下し、ランダムな角度（±15度など）で着地する演出。コミカルなインパクトに。
- **`make_wave_text`**: 文字がサイン波でゆっくり波打ちながら浮遊する（ゆらゆら）。夢の中や優しいシーン。
- **`make_shake_text`**: 文字が小刻みに震え続ける（ガタガタ）。恐怖や不安の表現。
- **`make_zoom_in_text`**: 巨大な文字が手前から叩きつけられるように着地（ドン！）。衝撃音やキメ台詞。
- **`make_glitch_text`**: 位置がランダムにズレるデジタルノイズ表現（ビリビリ）。SFやかっこいい系。
- **`make_float_in_text`**: 左から1文字ずつふわっと浮遊しながら出現（日常・優しい）。
- **`make_typewriter_text`**: 機械的にパパパッと1文字ずつ表示（タイプライター）。モノローグや説明文。
- **`make_step_text`**: 階段状（右肩上がり/下がり）にリズミカルに出現（ステップ）。前向きなサビに。
- **`make_heartbeat_text`**: 全体が「ドクン、ドクン」と鼓動する（ハートビート）。緊張や恋心。
- **`make_roll_in_text`**: くるくると回転しながら飛んでくる（ロールイン）。コミカルで賑やかなシーン。

## 3.6. 基本レイアウト・ルール (Standard Layout)
**視認性と映像のバランスを考慮し、以下の配置を基本とする。**

1.  **歌詞 (Lyrics)**:
    - **原則：画面下部 (Bottom)**
    - 中央 (Center) は映像の主題（キャラ等）と被るため、特別な演出（強調など）以外では避けること。
    - 目安: 画面下 15%〜20% の位置 (`y = H * 0.8`)

2.  **タイトル (Title)**:
    - **原則：画面やや上部 (Top-Center)**
    - **サイズ**: 歌詞よりも大きく設定すること (Title > Lyrics)。
    - 目安: 画面上 20%〜30% の位置 (`y = H * 0.2`)

3.  **作者名 (Author)**:
    - **原則：画面右下 (Bottom-Right)**
    - 謙虚かつクレジットらしく配置する。
    - 目安: 右端・下端からそれぞれ 5%〜10% の余白 (`x = W - w - margin`, `y = H - h - margin`)

# WORKFLOW STEP 4：SCENE-BASED DECISION RULES (Effect Mapping)
**演出の「強弱」をつけるため、カスタムライブラリと標準エフェクト（STEP 3）を適切に使い分けること。**

**基本方針:**
- **標準エフェクト (STEP 3)**: Aメロ、Bメロ、繋ぎのフレーズ、説明的なテロップ（過剰演出を避け、読みやすさ重視）
- **カスタムライブラリ (`mv_effects.py`)**: サビ、キメ台詞、感情のピーク、タイトル（ここぞという場面で投入）

---

- **静かなシーン / 日常 / 回想**:
  - 推奨（標準）: フェードイン, スライドイン, マスクイン
  - 推奨（カスタム）: `make_float_in_text` (ふわっと), `make_wave_text` (ゆらゆら)
  - 特徴: ゆったり、余白多め

- **盛り上がり（サビ） / 元気**:
  - 推奨（標準）: ポップイン, ビートスケール, ジャンプイン
  - 推奨（カスタム）: `make_bouncing_text` (PyonPyon), `make_zoom_in_text` (ドン！), `make_step_text`
  - 特徴: 動き大きく、中央配置

- **切なさ / 内省 / シリアス**:
  - 推奨（標準）: 一文字ずつ出現, 明滅（点滅）, ブラー
  - 推奨（カスタム）: `make_typewriter_text` (タイプライター), `make_heartbeat_text` (鼓動)
  - 特徴: 淡々と、または心理描写

- **激しい展開 / 恐怖 / 叫び**:
  - 推奨（標準）: シェイク, グリッチ（簡易）, 拡大
  - 推奨（カスタム）: `make_shake_text` (強震え), `make_glitch_text` (強ノイズ), `make_zoom_in_text`
  - 特徴: 不安定、インパクト重視

- **コミカル / 転調 / インパクト**:
  - 推奨（標準）: スライドイン（高速）, 回転
  - 推奨（カスタム）: `make_dropping_text` (落下), `make_roll_in_text` (回転飛来), ワープイン
  - 特徴: 意外性、物理挙動
