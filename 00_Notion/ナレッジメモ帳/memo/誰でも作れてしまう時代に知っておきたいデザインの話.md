## 1. 【目的提示（Why）】

この提案・説明の目的は、**生成 AI やノーコードツールで誰でも UI を作れる時代だからこそ、非デザイナーも理解しておくべき UI/UX デザインの基本原則を整理し、質の高いプロダクト体験を作るための視点を提供すること**です。

---

## 2. 【構造提示（How Many）】

記事内容は、大きく 7 つのポイントに分けられます。

1. メンタルモデルへのチューニング

1. ユーザー理解の重要性

1. 適切なフィードバック設計

1. 認知的負荷への配慮

1. 一度に見せる情報を絞る

1. 文字数の最適化

1. 一貫性のある UI 作り

---

## 3. 【論拠提示（What）】

**① メンタルモデルへのチューニング**

- ユーザーの「こう動くだろう」という直感（メンタルモデル）と、実装上の複雑さとのギャップを UI で埋める。

- 例：画像アップロードでは裏で複雑処理があっても、「押したらすぐ表示」することで期待通りの体験を実現。

**② ユーザー理解の重要性**

- 単なる簡略化は危険。特に専門職（医療・法律）では現場の知識や業務フローに合わせた UI が必要。

- MVP やプロトタイプで早期検証し、“無意識の行動”からニーズを発見できるのも現代の強み。

**③ 適切なフィードバック設計**

- 状態変化（完了・進行中・エラー）を適切に伝え、迷いをなくす。

- 過剰なフィードバックはストレスの原因になるため、量とタイミングの最適化が必要。

**④ 認知的負荷への配慮**

- 人間は同時に多くの情報を処理できないため、UI は情報整理と提示方法が重要。

- ステップごとに区切る、一問一答形式にするなどが有効。

**⑤ 一度に見せる情報を絞る**

- 一画面に詰め込みすぎず、7±2 チャンクを目安に情報を分割。

- Wantedly 募集作成のようにステップ化で集中度を上げる。

**⑥ 文字数の最適化**

- 読解は高負荷のため、必要な情報だけを簡潔に表示。

- 非言語（アイコン・色）で補完しつつ、初回体験ではあえて丁寧な説明も残す。

**⑦ 一貫性のある UI 作り**

- 同じ意味・機能の UI はスタイルを統一し、学習負荷を減らす。

- デザインシステムほど大規模でなくても、色・フォント・形状の共通化は有効。

---

## 4. 【網羅性確認（MECE）】

この 7 項目は、**UI/UX 設計の心理的・情報的・視覚的一貫性をカバーしており、抜け漏れが少ない構造**になっています。

追加考慮すべき視点としては：

- **アクセシビリティ**（高齢者や障害のあるユーザーへの配慮）

- **国際化対応**（多言語・文化圏での UI 適合）

- **継続改善プロセス**（ユーザー行動計測・AB テストによる改善サイクル）
