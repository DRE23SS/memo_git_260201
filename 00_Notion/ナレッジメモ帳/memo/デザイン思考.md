- なぜ、デザイン思考を学ぶ？

  - 昔：みんなの困りごとを解決する　 1→∞

    - 失敗しないように作る

  - 現代：物が溢れている
    - 物の改善よりも新規開発が求められる　 0→1
      - ストーリーに共感することで物が売れる

      - 速く失敗することが大事 → 試行錯誤

      - 好奇心駆動の問題解決

- 3 プロセスのサイクル

  - アイデア

    - 困りごとに共感し創作

    - 不安と共に固定概念を壊せるか＝ NOT ヒューリスティック

    - ==クリエイティブモード → 素材集め＆深く共感 → 発想==

    - 自分の枠にいる証拠（言葉）

      - 「こんなのでいいのかな・・・（妥協）」→TO BE:失敗を恐れない

      - 「それはちょっと・・・（違和感）」→TO BE:多様性・ユーザー視点

      - 「どうすればいいかな・・・（問題ありき）」→TO BE:0→1、もし、・・なら？

    - アイデアの種を見つける
      - 動作のバグ（無駄な行動）← 意識しないと意外と気が付かない

      - 愛のあるいちゃもん ← もっと、〇〇だと良い！「文句」＋「改善案」

      - 五感で考える ← まずは、口を動かす・手を動かす

  - 試作

    - 目的：アイデアを最短最速で体感する

    - コンセプトを創る

      - 誰に、何を

    - ストーリを創る

      - ユーザー中心で考える

      - 人は感情で動く

    - 絵で直感的に示す

    - 紙でプロンプトタイプを作成

    - 質問も試作の 1 つ

    - こだわりを磨く

      - どうやって、○○ を作るか？

      - どうやって、○○ にしていくか？

    - 自分の在り方を再定義
      - ○○ な △△ する体験 or 機会を提供する人　 ○○ ＝修飾語、△△ ＝動作

  - フィードバック
    - 次のアイデアに繋げる

    - 目的：仲間を作り、次の学びにつなげる

    - 受け手

      - ノリのいい人に FB をもらう

      - 目撃者（共同制作者）をつくる

      - 意図を伝えてアイデアを理解してもらう

    - 評価者

      - 批判しない（褒め＋改善案）

        - いいね！すごいね！天才かよ！（ダメと思ってもポジティブ基準で）

      - アイデアにフィードバックをする

      - ユーザー視点でフィードバック

    - ポイント
      - 見て議論する（視覚情報で伝える）

      - プラスに捉える＝挑戦＝成功/失敗が 50/50 が最低条件

      - 意味付けをする

- マインドセット（アイデア）

  - わくわく飛び出す（自分の枠から飛び出す）

    - 未来に面白いアイデアがある！

  - ==重要！：能動的に実験、やってみる！==

  - アイデアは赤ちゃん
    - アイデアは育てていくことで価値が生む！

- マインドセット（試作）
  - 試作は質問（うまくいくのか？）

  - 気楽にたくさん作ってみる

    - 色んな人の意見を聞いてみる

    - 未完成で良い

  - 失敗しても良いので量をこなして学ぶ
    - 失敗を判断する時間と質を考慮する

- A4 一枚思考法

  - ==頭の中を可視化できる・スイッチが入る・デザイン思考のプロセスと一致する==

  - A5 ノート・3 色ペン（視覚的トリガーを誘発）

  - 緑ペンで枠をつくる（見開きで 16 マス作る）

    - 左上のマスに日付とテーマ

  - 青ペン

    - 思いつくことを書きだす（3 分間）

    - 枠があることで強制力（埋めたくなる）

  - 赤ペン
    - 思考の絞り込み

    - 〇：頻度高い（3-5 個）

    - △：イライラ度高い（何個でも）

    - ■：重要度なもの（3-5 個）

- 体感ヒアリング＝弟子入り（現場を見て体感する）

  - 重視すること：ユーザーの声 ＜ 実体験

    - V ＝ f(x) 　※voice ＝ filter (experience)

  - ==チョークポイントは声ではなく気づきから発見できる！==

  - ユーザー＝師匠　自分＝弟子

    - ==実際に何をやっているかをそのまんま体験する！==

      - 師匠の話は要約されていて、不完全で例外を省いている

    - 教えてもらう → 深堀り → 確認 ↻ 教えてもらう

    - いい弟子入り＝「こんなんで良いの？」と言われること

    - やってはいけないこと

      - 次の質問を考える

      - 理解したつもり → 必ず確認する

      - 決めつけ＝解釈の押し付け

    - 避けるべき関係
      - 専門家

      - 現場視察

      - リサーチャー

  - 良くないヒアリング → 良いヒアリング

    - 話を掘り下げない → 文脈をクリックする

    - 早めの決めつけ → 深堀した内容をまとめる・確認する

    - 話を断りなく切り上げる → 深堀した内容をまとめる・確認する

  - ヒアリング対象は 3 人選ぶ → 多様性の受け入れ
    - メインターゲット

    - エクストリームユーザー（達人）

    - エクストリームユーザー（他人・クレーマー）

- 発想法（A4 思考法がベース）
  - 機能分解 → オズボーン

  - フィルターインプット → 掛け算を使用（A×X ＝？？）,浮かんだものは書きだして OK！

    - アイデア出しの段階で実現可能性は無視して OK！

  - アナロジー

    - A≒B 　違う（たとえ）世界線で考える

    - 図・絵で考える

  - 合体アナロジー
    - 機能分解 → 似た要素のモノを考える → 似たモノを機能分解 → フィルターインプット
